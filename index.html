The model was trained on Northwestern game data from the last 10 years. To test how well our model performed, we split all the data into the training set (70%) and the test set (30%). The data was randomly ordered, then split to ensure uniformity. This allowed us to get an estimate of how well our model would perform when given new data for a new season. Afterwards, we scaled the data and adjusted the means to make sure the learning algorithm effectively ran.
Our model was then trained using logistic regression with a binary classification output. The cutoff can be adjusted to change the amount of false positives and false negatives, but, rather than focus on that, we chose to maximize accuracy (found by dividing # of true positives by # of test data points).

After the model was trained, we tried adding regularization to increase the modelâ€™s performance on the test data set. However, due to the smaller dataset, we had relatively low variance. This meant that the model didnâ€™t struggle with overfitting.
